Row Context vs. Filter Context

Evaluation Logic & Propagation

PART 06

Row Context

"The Current Row"

Iterators & Columns

Refers to the current row during an iteration. It does not filter the model automatically, but allows you to reference columns for that specific row.

// Calculated Column or Iterator

SUMX( 'Sales', 'Sales'[Qty] * 'Sales'[Price] )

Context Transition

Triggered by
CALCULATE

Filter Context

"The Active View"

Measures & Visuals

The set of filters applied to the data model at evaluation time (via slicers, visuals, or DAX). It constrains the set of rows available for calculation.

// Measure calculation

CALCULATE( [Total Sales], 'Product'[Color]="Red" )

Relationship Propagation

DimTable
(One Side)
Filtered

Filters flow downhill
(1 to Many)

FactTable
(Many Side)
Restricted

Note: Row context does NOT propagate through relationships automatically. Only Filter context propagates.

Troubleshooting

  • Totals look wrong? Check for missing Filter Context.

  • Context Transition: Are you using measures inside columns?

  • Use ALL() or REMOVEFILTERS() to debug specific filters.
